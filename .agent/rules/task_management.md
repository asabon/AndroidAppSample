# タスク・Issue管理ルール

## プロジェクトの「唯一の真実（Source of Truth）」
- **docs/progress/task.md** を現在のプロジェクトステータスと常に一致させてください。
- すべてのタスク、未解決の課題、将来の展望はこのファイルに集約されます。

## タスクの追加と開始
- ユーザーから新しい要望（「～をやりたい」等）があった場合、まず **docs/progress/task.md** にタスクを追加し、全体像の中での位置付けを明確にしてください。
- 実装を開始する前に、新しく特定されたタスクに対して GitHub Issue を起票してください（/issue_new ワークフローの使用を推奨）。

## 作業中の進捗・反映
- 作業が進捗した際（コードの変更、問題の発見、議論による決定事項など）は、関連する GitHub Issue にコメントを残してください（/issue_comment ワークフローの使用を推奨）。
- 適宜 /save ワークフローを実行し、**docs/progress/task.md** の進捗状況やメモを最新に更新してください。

## タスクの完了
- タスクが完了し、coding.md に定められた検証をすべてパスした後は、以下の対応を行ってください：
  - **docs/progress/task.md** の当該項目を「完了」に更新する。
  - GitHub Issue をクローズする（または完了した旨をコメントする）。
